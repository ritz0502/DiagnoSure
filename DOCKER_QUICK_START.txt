## Troubleshooting

### "Requirements-ml.txt not found" error during build
**Fix:** Make sure the file exists in `backend/` directory:
```bash
ls backend/requirements*
# Should show: requirements-ml.txt  requirements.txt
```

### Build still slow (30+ minutes)
**Check:**
1. Verify PyTorch base image: `grep "pytorch/pytorch" backend/dockerfile`
2. Check Docker has cache enabled: `docker system df`
3. Last resort - clear cache: `docker system prune -a && docker-compose build`

### Container won't start
**Debug:**
```bash
docker-compose up --no-detach  # See real-time error messages
```

### Port 8090 already in use
**Fix:**
```bash
# Find what's using port 8090
netstat -ano | findstr :8090

# Or change docker-compose.yml:
# ports:
#   - "8091:8090"  <- Use 8091 instead
```
